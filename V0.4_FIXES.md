# ComfyUI-NG v0.4 — corrective release

This release fixes false model discovery and hardcoded defaults.

- No fabricated model catalogue.
- No `flux1-dev.safetensors` default in API, node catalogue, or frontend.
- The frontend selects the first model actually discovered on disk.
- Relative paths returned by the catalogue are resolved recursively under configured model roots.
- FLUX and Z-Image companion components may be fetched from their official Diffusers repositories unless `COMFYNG_OFFLINE=1`.
- Existing local Diffusers bundles are auto-discovered through `model_index.json`.
- `COMFYNG_COMPONENT_PATHS` adds extra component bundle roots.
- Krea2 remains dependent on a converted local Krea2 Diffusers directory because Diffusers currently documents no complete Hub repository for that pipeline.

The runtime never calls a legacy ComfyUI `/prompt` endpoint.
